### 1.2 Porter's 5 Forces — שוק שירותי פרויקטים טכנולוגיים בישראל

```
1. עוצמת התחרות — גבוהה
   ✦ עשרות dev houses, PM consultancies, product boutiques
   ✦ אבל: כמעט אף אחד לא מציע שלושתם תחת קורת גג אחת
   → GAP: End-to-End offering לא תפוס

2. איום כניסה חדשה — גבוה
   ✦ Barrier נמוך (הון, רישוי) — כל senior שעוזב חברה יכול לפתוח
   → חייבים לבנות brand moat מהר

3. כוח ספקים — נמוך
   ✦ ספקי עיקריים: Freelancers/subcontractors רבים בשוק
   → גמישות גבוהה בהרחבת קיבולת

4. כוח לקוחות — בינוני
   ✦ VC-backed companies: פחות price sensitive, יותר quality sensitive
   ✦ Bootstrapped: רגישים למחיר, קשה לסגור
   → ICP: חברות עם Runway ותקציב

5. תחליפים — גבוה
   ✦ In-house hiring (יקר ואיטי)
   ✦ Offshore/nearshore (זול אבל friction גבוה)
   ✦ Large integrators כמו Sapiens, Sela (יקרים ובירוקרטיה)
   → Positioning: מהיר מin-house, איכותי מoffshore, זמין מintegrators
```

### 1.3 TAM / SAM / SOM — חישוב Bottom-Up

```
TAM — שוק כולל שירותי IT/Tech בישראל: ~₪8 מיליארד
(גסה, Top-Down — לא מספיק)

SAM — חישוב Bottom-Up:
• חברות ישראליות 50-300 עובדים בענפי טכנולוגיה: ~3,000 חברות
• 30% מהן פעילות ב-Outsource Projects מדי שנה: ~900 חברות
• ממוצע הוצאה שנתית לפרויקטים חיצוניים: ₪200,000
→ SAM: ₪180,000,000

SOM — Year 1 (ריאלי):
• 10-12 חוזים × ₪25,000 ממוצע = ₪250,000-300,000
• 1-2 Retainers × ₪10,000/חודש × 6 חודשים ממוצע = ₪60,000-120,000
→ SOM Year 1: ₪300,000-420,000 (מטרה: ₪300K)
• Market Share: 0.17% מSAM — ריאלי לחלוטין
```

## חלק 6: LinkedIn B2B — הערוץ המרכזי

### למה LinkedIn ולא שום דבר אחר (קודם)

בישראל B2B:
- CTOs, VPs, CEOs ביחד עם עשרות אלפי פרופסיונלים — רובם על LinkedIn
- רשת מקצועית ישראלית צפופה → "אני מכיר אותו" קורה מהר
- Content Authority ב-LinkedIn אפשרי ב-90 יום עם consistency
- Organic Reach עדיין גבוה (בניגוד לFacebook/Instagram)

### מסלול A: Personal Brand של המייסד הראשי

**אופטימיזציית פרופיל (לפני פוסט ראשון):**
```
Headline: "Co-Founder @ Hipster Hippo | [Job] → [Outcome] | מוביל פרויקטים [End-to-End]"
About: 3 חלקים:
  1. מה עשית (הניסיון)
  2. מה אתה עושה עכשיו ולמה
  3. עם מי הכי טוב לדבר (ICP direct)
Featured: 3 פריטים — פוסט הכי חזק / Website / Case Study (כשיהיה)
```

**Content Mix שבועי (3-4 פוסטים):**

| יום | פורמט | נושא |
|-----|--------|-------|
| א' | Text + Image | תובנה מניסיון: "אחרי 10 שנה ב[domain], הנה מה שחברות מפספסות..." |
| ג' | Carousel / Document | Process/Framework: "5 שאלות ששואלים בKickoff כל פרויקט" |
| ה' | Short Text | Hot Take: "דעה שנויה במחלוקת על [topic] בענף" |
| ו' (אופציונלי) | Behind the Scenes | "בנינו את Hipster Hippo — הנה מה שלמדנו השבוע" |

**שלב 3: Outreach Sequence**
```

  "שמתי לב לפוסט שלך על [X] — מסכים לגמרי/מחשבה מעניינת.
   מחבר בין אנשים שמובילים פרויקטים טכנולוגיים."

שבוע 2 (אחרי Connect): Value Message (לא Pitch!)
  "שתף ב-Feed שלי [article/framework] שחשבתי שיכול לעניין אותך
   בהקשר של [מה שראיתי שעוסקים בו]."
```

- Template גנרי בלי personalization = מחיקה

## חלק 7: Content Strategy — Zero Case Study Framework

### הבעיה: אין Case Studies
### הפתרון: The Expertise-First Method

```
במקום: "הנה מה שבנינו ללקוח X"
תוציאו: "הנה ה-Framework שאיתו בנינו פרויקטים כמו X"

במקום: "הנה Case Study שלנו"
תוציאו: "הנה מה שלמדנו מ-N שנות ניסיון בפרויקטים האלה"

ה-Expertise מהתעסוקה הקודמת = התחליף הלגיטימי לPortfolio.
```

### 4 Content Pillars

```
PILLAR 1 — "THE VAULT" (60% מהתוכן)
מה: ידע עמוק שצוברים ב-N שנה בתעשייה
דוגמאות:
- "המדריך המלא לScoping פרויקט — מה לשאול לפני שחותמים"
- "5 סיגנלים שMVP שלך לא מוכן לScale"
- "איך לנהל CTO חיצוני — 8 כללים שיצילו אתכם"
מטרה: מוכיח Expertise ללא Case Studies

PILLAR 2 — "THE PROCESS" (20%)
מה: שקיפות על אופן העבודה של Hipster Hippo
דוגמאות:
- "ככה נראה Discovery Sprint אצלנו — שלב-אחרי-שלב"
- "הShort-List שלנו לפני שאנחנו אומרים כן לפרויקט"
- "מה קורה בWeekly Client Update שלנו"
מטרה: מוריד Risk תפיסה מלקוחות שלא מכירים אתכם

PILLAR 3 — "THE STORY" (10%)
מה: למה הקמנו, מה אנחנו מאמינים, מה הדרך
דוגמאות:
- "הקמנו Hipster Hippo אחרי X שנה בענף — הנה למה"
- "מה שנשבר בDev Houses הגדולים שהכרחנו אותנו להקים משהו חדש"
- "Week 4 — מה בנינו, מה שברנו, מה למדנו"
מטרה: Connection אמוציונלי + Trust

PILLAR 4 — "THE PROOF" (10%)
מה: ברגע שיש לקוח ראשון — הכל יוצא מהר
דוגמאות:
- "Milestone ראשון — לקוח חתם" (ללא שם, עם Context)
- "Mini Case: פרויקט שהחל ב-Discovery Sprint → איך הגיע לDelivery"
- Testimonial quote מלקוח (ברשות)
מטרה: Social Proof ראשוני
```

## חלק 8: CRM וניהול Pipeline

### Stack מינימלי (שלב 1)

```
CRM: HubSpot Free

Scheduling: Calendly Free
Proposals: Notion / Google Docs (+ PDF)
Contracts: PandaDoc Free / DocuSign Starter
Tracking: UTM Links + GA4 (כשיש אתר)
```

### Deal Stages + הגדרה

```
1. CONVERSATION    → שיחה ראשונית נוצרה (Discovery Call לא נקבע עדיין)
2. DISCOVERY CALL  → Meeting קבוע
3. QUALIFIED       → Fit אושר (תקציב, זמן, ICP match)
4. PROPOSAL SENT   → הצעת מחיר נשלחה
5. NEGOTIATION     → משא ומתן פעיל
6. WON             → חתמו ✅
7. LOST            → לא סגרנו (חובה לתעד: למה?)
```

### Pipeline Review שבועי (30 דקות, חובה)

```
□ מה זז מStage לStage?
□ מה תקוע? מה הCTA הבא?
□ מה נסגר/אבד השבוע?
□ כמה Conversations חדשות נפתחו?
□ Pipeline Value סה"כ vs. Target
```

### Nurture לSales Cycle ארוך (45-120 יום)

```
WARM Leads (שיחה הייתה, לא מוכנים עדיין):
□ LinkedIn: עקוב אחריהם, הגב לתוכן שלהם
□ Newsletter חודשי: תמיד "Reply" אופציה אישית
□ Check-in ידני כל 6-8 שבועות: "הכל בסדר? ראיתי שהחברה שלך..." (Trigger)

COLD Leads (מכירים, לא הייתה שיחה):
□ Newsletter בלבד עד שיש Trigger
□ Trigger Events: Job Change, Company News, LinkedIn Post רלוונטי
```

### Lead Scoring (פשוט לצוות קטן)

```
🔴 HOT → פעולה השבוע:
   שיחה התקיימה + צורך ספציפי + תקציב הוזכר + Timeline ברור

🟡 WARM → Nurture שבועי:
   שיחה הייתה, עניין קיים, לא מוכן עדיין

⚪ COLD → Newsletter + מעקב חודשי:
   מכיר את המותג, אין שיחה

❌ DISQUALIFIED → Suppress:
   ICP Mismatch, תקציב לא מתאים, מתחרה
```
